# PRD: 학원 출석/숙제/칭찬 스티커 랭킹 앱

버전: v2.0 · 작성일: 2026-07-19 · 작성자: Sabrina

### v2.0 변경 요약 (v1.1 → v2.0)
- **스티커 정책**: 출석 지급 기준을 5단계(정시/10분/30분/1시간/1시간 초과)로 고정하고, 반별로는 **정규 출석 시각만** 조정 가능하도록 변경 (지급 개수는 전역 고정)
- **랭킹**: 상대 비율(%) 방식을 폐지하고 **1등=금, 2등=은, 3등=동 각 1명 고정** 방식으로 변경, 포디움(1-2-3위 카드) UI 도입
- **반 관리**: 반 관리 탭에 **+ 버튼**으로 반을 자유롭게 추가/생성 가능(더 이상 5개 고정 아님)
- **반 소속**: 학생이 반을 직접 선택해 **승인요청**하고, 관리자가 승인해야 소속되는 구조로 변경(학생 앱 설정 화면에서 토글 선택 → 승인요청)
- **랭킹 노출 설정**: 기본 노출 단위기간(이번 달), 일/주/월/분기 단위 선택, **그룹(반)별 랭킹 우선 노출** 디폴트, 그룹별 랭킹 단위기간 개별 설정
- **상품(리워드)**: 랭킹 노출 설정과 연동 — 그룹별·주기별로 보상을 **개별 설정**해야 하며, 주기가 끝나면 자동 갱신되지 않고 관리자가 다음 주기 보상을 다시 세팅
- **온보딩**: 학생 가입 시 학원 이름 · 나이 입력
- **멀티테넌시**: 선생님(관리자)은 여러 명 존재 가능하며 각자 독립된 학원(테넌트) 운영, 선생님 하위에 **조교(보조 선생님)** 위임 가능, **초대 링크**를 통한 학생 가입(링크 발급 선생님의 학원·반에 자동 소속)
- **기술 스택**: 반응형 웹앱, Vercel 배포, Supabase(Auth/DB/Storage) 연동

> IA(정보구조) 상세는 별도 파일 `IA_학원스티커랭킹앱.md`, 디자인 가이드는 `디자인가이드_학원스티커랭킹앱.md` 참고.

## 1. 개요

### 1.1 배경 및 목적
학원(들)이 각자 독립적으로 사용하는 **멀티테넌트 SaaS 형태의 출석/숙제/칭찬 스티커 랭킹 앱**. 학생은 자기 계정으로 로그인해 스티커 현황과 랭킹을 확인하고, 선생님(관리자)은 스티커 지급 기준·반·랭킹 주기·보상을 운영하며 모든 지급 내역을 감사·롤백할 수 있다. 서로 다른 학원의 데이터는 완전히 분리되며, 학생은 특정 선생님이 발급한 초대 링크를 통해 해당 학원에 가입한다.

### 1.2 대상 사용자 (역할 3단계)
- **원장/선생님 (Owner)**: 학원 단위 전체 관리 권한. 조교 초대, 학생 초대 링크 발급, 반/정책/랭킹/보상 전체 설정.
- **조교 (Assistant)**: Owner가 위임한 보조 관리자. 반/학생/승인/로그/보상 등 일상 운영 권한을 가지되, 다른 선생님(조교) 관리나 학원 자체 설정 변경은 불가.
- **학생 (Student)**: 초대 링크로 가입, 자기 계정으로 로그인해 출석·숙제·칭찬 스티커 활동 및 반 신청, 랭킹·보상 확인.

### 1.3 범위 (In Scope)
멀티테넌트 온보딩(역할 구분, 초대 링크, 학원 가입), 조교 위임, 출석/숙제/칭찬 스티커(기존 로직 유지 + 정책 단순화), 반 동적 생성 및 학생 자율 신청·승인, 스티커 지급 로그·롤백, 랭킹(1/2/3등 고정, 그룹 우선 노출, 노출 주기 설정), 랭킹 주기와 연동된 그룹별 상품(리워드) 운영, 공지사항, 반응형 웹 UI.

### 1.4 범위 외 (Out of Scope, v2.0)
결제/구독 과금, 학부모 계정, 실시간 채팅/1:1 상담, 네이티브 앱(iOS/Android 스토어 배포) — 반응형 웹앱으로 대응.

## 2. 용어 정의

| 용어 | 정의 |
|---|---|
| 학원(테넌트) | 하나의 원장 선생님을 중심으로 운영되는 독립된 조직 단위. 학원 간 데이터는 서로 보이지 않음 |
| 원장(Owner) | 학원을 개설한 최초 관리자. 조교 위임, 초대 링크 발급 등 모든 권한 보유 |
| 조교(Assistant) | 원장이 추가한 보조 관리자. 일상 운영 권한 위임받음 |
| 초대 링크 | 특정 선생님(원장/조교)이 발급하는 학생 가입용 링크. 이 링크로 가입한 학생은 해당 학원 + 발급자의 기본반에 자동 소속 |
| 기본반 | 학원 내 모든 학생이 자동 소속되는 상시 반 |
| 특강반(그룹) | 학생이 직접 신청 후 승인받아 소속되는 반. 운영 기간이 있으며 종료 시 자동 해제 |
| 반 승인요청 | 학생이 특정 반 소속을 신청하면 관리자 승인이 있어야 실제 소속되는 절차 |
| 랭킹 단위기간 | 랭킹을 집계하는 기준 주기. 일/주/월/분기 중 선택. 전체 랭킹과 그룹별 랭킹에 각각 설정 가능 |
| 그룹 우선 노출 | 학생이 소속 그룹(반)이 있는 경우, 홈 화면 랭킹을 전체가 아닌 그룹 랭킹부터 기본 노출하는 정책 |
| 포디움(Podium) | 1·2·3등을 카드 형태로 시각화한 랭킹 상단 영역(금/은/동, 각 1명) |
| 롤백(취소) | 관리자가 부정/오류로 판단한 지급 건을 무효화하는 조치. 이력은 보존, 취소 상태로 표시 |

## 4. 핵심 기능 명세

### 4.1 온보딩 & 계정
- 가입 시작 시 **"선생님으로 시작하기" / "학생으로 시작하기"** 중 역할을 선택한다.
- **선생님(Owner) 가입**: 이메일/비밀번호 등 계정 생성 → 학원 이름 입력 → 학원(테넌트) 생성 → 기본반 자동 생성.
- **학생 가입**: 반드시 선생님이 발급한 **초대 링크**를 통해 접근한다. 링크에는 `academy_id`(학원)와 `inviter_teacher_id`(발급자)가 포함되어 있어, 가입 완료 시 해당 학원 소속 + 발급자 정보가 자동 연결되고 **기본반에 자동 배정**된다.
  - 가입 입력 항목: 이름, 나이, (학원 이름은 링크로 자동 연결되어 확인만 가능, 별도 입력 불필요), 로그인 정보(이메일/비밀번호 또는 소셜 로그인)
  - 초대 링크 없이 접근한 경우 가입을 진행할 수 없고, "선생님에게 초대 링크를 요청하세요" 안내를 노출한다.
- 가입 완료 후 학생 앱 설정 화면에서 언제든 소속 학원명·담당 선생님 이름을 확인할 수 있다.

### 4.2 멀티테넌시 & 조직 구조
- 학원(테넌트) 단위로 모든 데이터(학생, 반, 스티커 로그, 랭킹, 보상, 공지)가 완전히 분리된다. A학원 선생님의 화면에는 A학원 학생만 보인다.
- 원장은 조교를 추가해 운영을 위임할 수 있다. 조교는 원장과 동일하게 반/학생/승인/로그/보상/공지 운영이 가능하나, **다른 선생님(조교) 관리, 학원 자체 설정(학원명 변경 등)은 원장만 가능**하다.
- 초대 링크는 원장·조교 모두 발급 가능하며, 발급자 정보가 학생 레코드에 남아 "어느 선생님이 초대했는지" 추적할 수 있다.
- 초대 링크는 학원당 여러 개 발급 가능(예: 반별/시기별로 별도 링크 운용). 링크 만료/재발급 정책은 Open Question 참고.

### 4.3 출석 스티커 (정책 단순화)
출석 지급 스티커 수는 **5단계 전역 고정값**이며 관리자가 변경할 수 없다. 관리자가 반별로 조정할 수 있는 값은 **정규 출석 인정 시각(=출석 시간)** 하나뿐이다.

| 구간 | 지급 스티커 (고정, 수정 불가) |
|---|---|
| 정시 이전 출석 | 5장 |
| 정시 후 10분 이내 | 4장 |
| 정시 후 30분 이내 | 3장 |
| 정시 후 1시간 이내 | 2장 |
| 정시 후 1시간 초과 | 1장 |
| 결석 | 0장 |

- 스티커 정책 설정 화면에는 이 5단계 표를 **읽기 전용 안내**로만 노출한다.
- 반 관리 화면에서 반마다 "정규 출석 시각"(예: 대수반 19:00, 수능반 09:00)만 입력하면, 위 5단계 판정은 그 시각을 기준으로 자동 적용된다.

### 4.4 숙제 스티커 (변경 없음)
| 완료율 | 지급 스티커 |
|---|---|
| 완료 (100%) | 5장 |
| 절반 완료 (50%) | 3장 |
| 미완료 (0%) | 0장 |

학생이 완료율을 선택해 인증 신청 → 관리자 승인 필요. 지급 수는 관리자가 정책 설정에서 조정 가능(기존과 동일, 3단계 유지).

### 4.5 칭찬 스티커 (변경 없음)
학생이 사유를 적어 요청 → 관리자가 지급 수를 입력해 승인. 관리자가 요청 없이 직접 지급도 가능. 카테고리 구분은 관리자 조회용으로만 사용, 랭킹은 항상 전체 합산 기준.

### 4.6 반 관리 & 반 소속 승인 플로우
- **반 추가**: 반 관리 탭 상단 **"+ 반 추가"** 버튼으로 관리자가 새 반을 생성한다. 입력 항목: 반 이름, 정규 출석 시각, 운영 기간(특강 기간, 기본반은 상시로 기간 없음).
- 반은 더 이상 고정된 5개(기본/대수/수2/미적/수능)가 아니라 **관리자가 자유롭게 추가·수정·종료**할 수 있는 동적 목록이다.
- **학생 반 신청**: 학생 앱 설정 → 프로필 설정 → "소속 반 신청하기" 버튼 → 반 목록이 **토글(다중 선택)** 형태로 표시 → 원하는 반을 선택 후 **"승인요청하기"** 버튼 → 각 반에 대해 승인 대기(pending) 신청이 생성된다.
- **관리자 승인**: 승인 대기 신청은 **학생 관리 탭**과 **반 관리 탭**(해당 반 상세) 양쪽에서 모두 확인 가능하며, 승인/반려 처리한다. 승인 시 학생이 실제로 그 반에 소속되고, 반려 시 신청이 기각된다.
- 기본반은 초대 가입 시 자동 배정되며 승인 절차가 필요 없다.
- 특강반 운영 기간이 종료되면 소속이 자동 해제된다(기존과 동일).

### 4.7 스티커 승인 및 롤백/감사 로그 (변경 없음)
모든 스티커 지급은 지급 로그(학생/유형/개수/사유/처리자/시각/상태)로 남으며, 관리자는 부정 지급을 확인해 롤백(취소)할 수 있다. 취소된 로그는 삭제되지 않고 이력으로 보존되며 랭킹/총합에서 제외된다.

### 4.8 랭킹 시스템 (등급 방식 변경)
- **등급 산정**: 비율(%) 방식을 폐지하고, **1등 = 금, 2등 = 은, 3등 = 동을 각각 정확히 1명씩** 부여한다. 4위 이하는 순위 숫자만 표시한다.
  - 참가자가 3명 미만인 그룹은 존재하는 순위까지만 메달을 부여한다(예: 2명뿐이면 금/은만 존재).
  - **동점 처리(기본 정책)**: 스티커 총합이 동일한 경우, 해당 총합에 먼저 도달한 학생(스티커 획득 시각 기준 빠른 순)이 상위 순위를 가진다. 세부 규칙은 Open Question 참고.
- **UI**: 랭킹 화면 상단은 **포디움 형태**로 표시한다 — 2등(좌, 중간 크기) · 1등(중앙, 가장 크게 + 왕관 아이콘) · 3등(우, 중간 크기) 카드를 나란히 배치하고, 그 아래 4위부터는 리스트 형태로 순위·이름·스티커 수를 표시한다. (상세 스펙은 디자인 가이드 참고)
- **범위**: 전체 랭킹 + 그룹(반)별 랭킹. 학생은 자신이 속한 그룹들 + 전체 랭킹을 볼 수 있다.
- **집계 기준**: 취소되지 않은 스티커(출석+숙제+칭찬) 합산.

### 4.9 랭킹 노출 설정
- **기본 노출 단위기간의 기본값**: 오늘 날짜가 속한 **이번 달(해당 월 1일~말일)**.
- **단위 선택**: 관리자는 랭킹 집계 단위를 **일 단위 / 주 단위 / 월 단위 / 분기(4분기) 단위** 중에서 선택할 수 있다. 단위를 바꾸면 노출 기간이 해당 단위의 현재 주기로 자동 계산된다(예: 주 단위 선택 시 이번 주 월~일).
- **그룹 우선 노출(디폴트)**: 학생이 하나 이상의 그룹(특강반)에 소속된 경우, 홈 화면 랭킹 블록은 **전체 랭킹이 아닌 그룹 랭킹을 기본으로 우선 노출**한다. 여러 그룹에 속한 경우 가장 먼저 승인된 그룹을 기본값으로 하고, 학생은 탭으로 다른 그룹/전체 랭킹으로 전환할 수 있다. 그룹이 없는 학생(기본반만 소속)은 전체 랭킹을 기본 노출한다.
- **그룹별 랭킹 단위기간 개별 설정**: 랭킹 노출 설정 탭에 **"그룹별 랭킹 단위기간 설정하기"** 버튼을 두어, 그룹(반)마다 서로 다른 단위기간(예: 대수반은 주 단위, 수능반은 월 단위)을 설정할 수 있다. 전체 랭킹은 별도의 전역 단위기간을 갖는다.

### 4.10 상품(리워드) 시스템 — 랭킹 주기 연동
- 보상(리워드) 캠페인은 **특정 그룹 + 그 그룹에 설정된 랭킹 단위기간**에 귀속되어 설정된다. 예: 대수반의 랭킹 단위가 "주 단위"라면, 대수반 보상도 매주 단위로 설정해야 한다.
- **단위기간이 종료되면 해당 캠페인은 자동으로 "종료" 상태가 되며, 자동으로 다음 주기에 재사용되지 않는다.** 관리자는 새로운 주기가 시작될 때마다 **그 그룹의 보상을 다시 설정**해야 한다(자동 롤오버 없음 — 의도된 설계로, 매 주기 순위가 초기화되므로 보상 내용도 매번 재검토 필요).
- 상품 등록 화면에서 캠페인 생성 시 "적용 그룹"을 선택하면, 그 그룹에 설정된 랭킹 단위기간이 자동으로 표시되어 기간을 그대로 사용하거나 수동으로 조정할 수 있다.
- 순차 선택(드래프트) 방식, 대상 비율/인원 설정, 이미지·링크 등록 등 기존 기능은 유지.

### 4.11 공지사항 (v1.1 유지)
관리자 게시판에서 작성한 공지가 학생 홈 화면 상단 플랩 배너에 순환 노출. 고정 공지 우선.

## 5. 주요 사용자 시나리오

1. A학원 원장이 가입해 학원을 개설하고 조교 1명을 초대 → 조교가 학생 초대 링크를 발급 → 학생이 링크로 접속해 이름/나이를 입력하고 가입 → A학원 기본반에 자동 소속.
2. (대수반은 이미 조교가 개설해둔 상태) 학생이 설정 화면에서 "대수반"을 토글로 선택해 승인요청 → 조교가 학생 관리 탭에서 요청을 확인하고 승인 → 학생이 대수반 랭킹에 노출되기 시작.
3. 학생이 대수반 정규 출석 시각(19:00) 이전에 출석 체크 → 5장 즉시 지급(5단계 고정 기준 적용).
4. 관리자가 반 관리 탭에서 "+ 반 추가"로 "물리 특강반"을 새로 생성하고 정규 출석 시각을 20:00으로 설정.
5. 관리자가 랭킹 노출 설정에서 대수반의 랭킹 단위를 "주 단위"로 설정 → 이번 주 랭킹이 즉시 계산되어 학생 홈 화면(그룹 우선 노출)에 표시.
6. 관리자가 대수반의 이번 주 보상을 등록(상위 3명 대상) → 주가 끝나면 캠페인이 자동 종료 → 다음 주 보상은 관리자가 다시 등록.
7. 학생 홈 화면 랭킹 블록에 1등(금)·2등(은)·3등(동) 포디움과 4~5위 리스트 노출, 전체보기로 전체 랭킹 확인.

## 6. 화면 요구사항

### 6.1 온보딩
- 역할 선택(선생님/학생)
- 선생님: 계정 생성 → 학원명 입력 → 완료
- 학생: 초대 링크 접속 → 이름/나이 입력 → 계정 생성 → 완료(학원/기본반 자동 표시)

### 6.2 학생 화면 (v1.1 구조 유지 + 변경 사항 반영)
- 상단바: 로고, 알림, 설정
- 홈: 공지 배너, 이용자 정보, **랭킹 포디움(그룹 우선 노출 디폴트, 탭으로 전체/다른 그룹 전환)**, 총 스티커 블록(체크 CTA), 상품 블록
- 스티커 탭: 출석(반별 시각만 다름, 5단계 안내는 아코디언) / 숙제(3단계) / 칭찬(요청내역 아코디언)
- 마이페이지: **소속 반 관리(반 신청은 이제 설정 화면으로 이동, 마이페이지에서는 현재 소속/신청 상태 조회)**, 스티커 이력(아코디언)
- 설정: 소속 학원명, 담당 선생님 이름, **프로필 설정(사진 등록, 이름/나이 수정)**, **소속 반 신청하기(토글 다중 선택 → 승인요청하기 버튼)**, 데모 계정 전환(개발용)

### 6.3 관리자 화면
- 로그인/온보딩
- 대시보드
- 공지사항 게시판
- 스티커 정책 설정: 출석 5단계 지급 기준(읽기 전용 안내) + 숙제 3단계 지급 수(수정 가능)
- 반 관리: **+ 반 추가 버튼**, 반별 정규 출석 시각 설정, **반별 승인 대기 신청자 목록**
- 학생 관리: 학생 목록, **반 승인 대기 요청 처리(승인/반려)**
- 승인함: 숙제/칭찬 요청 승인·반려
- 스티커 로그·감사: 필터, 롤백
- **랭킹 노출 설정**: 기본 단위기간(이번 달 디폴트), 단위 선택(일/주/월/분기), 그룹 우선 노출 안내, **그룹별 랭킹 단위기간 설정하기**
- **상품(리워드) 관리**: 그룹 + 해당 그룹 랭킹 주기 연동, 주기 종료 시 자동 종료 처리
- 조직 관리(신규): **조교 추가/권한 관리**, **학생 초대 링크 발급/목록**

## 9. 기술 스택 & 아키텍처
- **프론트엔드**: 반응형 웹앱(모바일/태블릿/데스크톱 대응), Next.js 등 React 기반 프레임워크 권장

- **인증**: 이메일/비밀번호 또는 소셜 로그인(추후 확정), 역할(owner/assistant/student) 기반 접근 제어 + RLS
- **초대 링크**: 토큰 기반 URL(`/invite/{token}`), 서버에서 tenant_id·issuer 검증 후 가입 처리

## 10. 비기능 요구사항
- 학원(테넌트) 간 데이터는 RLS 등으로 완전히 격리되어야 함
- 스티커 지급/취소, 반 승인/반려는 즉시 랭킹에 반영(실시간/준실시간)
- 모든 지급/취소/승인 행위는 변경 불가능한 이력으로 보존
- 반, 랭킹 정책, 랭킹 단위기간, 보상 등은 관리자가 코드 배포 없이 화면에서 설정 가능해야 함
- 특강반 종료일 도래 시 자동 소속 해제, 랭킹 주기 종료 시 보상 캠페인 자동 종료 처리(스케줄 처리 필요)
- 반응형 레이아웃으로 모바일/데스크톱 모두 대응
- 디자인 에셋은 `디자인가이드_학원스티커랭킹앱.md` 기준으로 적용
