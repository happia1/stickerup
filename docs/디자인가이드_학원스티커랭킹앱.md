# 디자인 가이드: 학원 출석/숙제/칭찬 스티커 랭킹 앱

버전: v2.0(블랙 기본 테마) · 개정일: 2026-07-20 · 참고: 사용자 첨부 레퍼런스 스크린샷(랭킹 포디움 화면, 성취 팝업, 선택형 팝업, 탭/더보기/가로카드 화면)

이 문서는 첨부된 레퍼런스 앱의 UI 패턴(포디움형 랭킹보드, 팝업 구조, 버튼, 내비게이션, 펼치기/접기, 가로 스크롤 카드)을 우리 앱의 톤에 맞게 재해석한 디자인 시스템이다. v2.0부터 **모든 화면의 기본 표면을 블랙 계열 다크 테마로 통일**했다(구 v1.0의 "라이트 기본 + 다크는 축하 화면 전용" 이원화 원칙은 폐기됨). 디자인 에셋(일러스트, 캐릭터 이미지 등)은 사용자가 직접 제작해 적용할 예정이므로, 여기서는 **레이아웃 구조·타이포·컬러·상태 스펙**을 규정한다.

---

## 1. 디자인 원칙
- **블랙 기본 표면**: 모든 화면(홈, 스티커, 마이페이지, 리스트, 설정, 관리자 화면 포함)은 블랙 계열 다크 표면을 기본으로 한다. 라이트 표면 예외는 없다.
- **명도 단계로 위계 표현(Elevation)**: 표면은 3단계 명도 스케일로 위계를 나타낸다 — `surface-page`(가장 어두움, 화면 바탕) < `surface-card`(카드/리스트, 1단계 상승) < `surface-raised`(팝업·포디움·사이드바 등 가장 강조되는 요소, 2단계 상승). 밝을수록 사용자와 가까운/강조되는 레이어라는 의미다.
- **특별한 순간의 강조**: 랭킹 포디움, 보상 획득, 배지 획득 팝업 등 축하·성과 순간은 `surface-raised`를 사용해 주변 화면보다 한 단계 더 밝게 띄워 "떠오르는" 느낌으로 대비를 만든다(과거처럼 라이트/다크를 뒤바꾸는 대신, 같은 다크 팔레트 안에서 명도 대비로 특별함을 표현).
- **한 화면 한 강조색**: 화면마다 강조색(Primary)은 앰버 1개만 사용. 뱃지/등급색(금·은·동)은 의미 전달용으로만 예외 허용. 블랙 배경 위에서는 앰버(#FF9F1C)가 이미 충분한 대비를 가지므로 별도의 "진한 앰버"를 쓰지 않는다.
- **정보 우선순위**: 큰 숫자(스티커 수, 금액성 데이터) → 중간 굵기 타이틀 → 보조 설명 순으로 시각적 위계를 분명히 한다.
- **터치 타깃 최소 44×44px**, 카드 코너는 12~16px 라운드로 통일.

## 2. 컬러 토큰

| `--surface-raised` | #20232B | 팝업, 랭킹 포디움, 관리자 사이드바, 성취 모달(2단계 상승 — 가장 강조) |
| `--border` | #2A2D35 | 카드/구분선 |
| `--border-strong` | #383C46 | 강조가 필요한 구분선 |
| `--text-primary` | #F5F6F7 | 본문/제목 |
| `--text-secondary` | #9AA0AA | 보조 설명 |

## 3. 타이포그래피 스케일
폰트: 시스템 기본(Apple SD Gothic Neo / -apple-system / Segoe UI 순 폴백). 굵기는 400(Regular)/600(Semibold)/800(Bold) 3단계만 사용.

| 레벨 | 크기 | 굵기 | 용도 |
|---|---|---|---|
| Display | 28px | 800 | 랭킹 포디움 1등 이름, 보유 총 스티커 수 등 히어로 숫자 |
| Title | 19~20px | 800 | 화면 제목(예: "어떤 맛을 고를까요?") |
| Subtitle | 15px | 700 | 카드 제목, 섹션 헤더 |
| Body | 13.5~14px | 400 | 본문, 리스트 항목 |
| Caption | 12px | 400 | 보조 설명, 날짜 |
| Micro | 11px | 600 | pill·뱃지·태그 텍스트 (최소 크기, 이 이하 금지) |

행간(line-height)은 본문 1.5, 제목 1.3을 기본으로 한다. 텍스트 컬러는 `--text-primary`(#F5F6F7) 고정 사용을 기본으로 하며, 절대 순백(#FFFFFF) 고정 사용 금지(표면과 마찬가지로 텍스트도 브랜드 톤이 섞인 오프화이트를 사용).

## 4. 랭킹 보드(포디움) 스펙
레퍼런스의 "2위-1위-3위 카드 + 4위 이하 리스트" 구조를 그대로 채택한다.

- **배경**: `--surface-raised` (포디움 영역만; 전체 화면이 아니라 랭킹 블록/오버레이 내부에 한정 적용 — 주변 카드보다 한 단계 밝혀 "특별한 순간"을 표현)
- **상단 3열 그리드**: 2등(좌) · 1등(중앙) · 3등(우). 1등 카드는 좌우보다 **높이 +12~16%, 폭 +8~12%** 크게 표시해 시각적 정점을 만든다.
- **각 카드 구성 요소(위→아래)**: 순위 왕관/뱃지(1등만 금색 크라운 아이콘, 2·3등은 원형 숫자 뱃지) → 아바타/프로필 이미지(원형, 1등 56px / 2·3등 44px) → 이름(Subtitle, text-primary) → 스티커 수(Body, brand-amber, 굵게) → 소속 반 pill(Micro, surface-card 배경)
- **카드 배경색**: 1·2·3등 카드 모두 `--surface-card`로 통일(포디움 바탕인 `--surface-raised`보다 한 단계 어둡게 눌러 앉은 느낌을 주어 개별 카드를 구분 — 레퍼런스처럼 카드별 임의 색상 대신 뱃지 색으로만 등급 구분, 등급색이 곧 정보이므로 배경색 낭비 금지)
- **4위 이하 리스트**: 좌측 순위 숫자(원형 배지, 배경 `--surface-card`) → 아바타(32px) → 이름 → 우측 스티커 수. 행간 구분선 `--border` 1px.
- **참가자 3명 미만 그룹**: 존재하는 순위까지만 카드 렌더링, 빈 슬롯은 표시하지 않음(빈 카드 X).
- **동점 처리 UI**: 동일 스티커 수라도 카드/리스트에는 확정된 단일 순위만 노출(공동 순위 표기 없음) — 내부 tie-break 규칙은 PRD 참고.

## 5. 팝업(모달) 스펙

### 5.1 성취/축하 팝업 (예: 보상 획득, 랭킹 진입 알림)
레퍼런스의 "오늘 참여자 N명 중 상위 1%" 팝업 구조 채택.
- 배경: 화면 전체에 `rgba(0,0,0,.65)` 딤 처리 + 팝업 카드는 `--surface-raised`, 코너 20px 라운드
- 구성(위→아래, 모두 중앙 정렬): Caption(참여 규모 등 맥락 문구, text-secondary) → Title(성과 문구, text-primary, 800) → 중앙 아이콘(80~96px, 상징 아이콘 예: 크라운/트로피/스티커) → Body(축하 메시지 1줄, text-secondary) → Primary CTA 버튼(전체 너비 - 좌우 24px 여백)
- 팝업은 자동으로 뜨는 트리거(예: 랭킹 최초 진입, 보상 확정) 시에만 사용하고 남용하지 않는다.

### 5.2 선택형 팝업 (예: 상품 선택, 반 신청)
레퍼런스의 "아이스크림 맛 고르기" 화면 구조 채택 — 우리 앱의 **상품(리워드) 선택**, **반 신청 토글** 화면에 그대로 적용.
- 배경: `--surface-page`(다른 실무 화면과 동일한 기본 다크 표면 사용 — v2.0부터 선택형 팝업도 축하 팝업과 같은 다크 팔레트를 공유하되, `--surface-raised`를 쓰지 않아 톤 차이로 구분)
- 상단 내비게이션: 좌측 back 화살표 아이콘 버튼, 우측 텍스트 링크(예: "안내", "자주하는 질문") — 3.7절 참고
- Title(20px/800) + Subtitle(13px/400, text-secondary) 2줄
- **선택 그리드**: 2열(모바일 기준) 카드, 카드당 이미지/아이콘 + 라벨. 선택 시 `border: 2px solid var(--brand-amber)` + 배경 `--warning-bg`(#3A2A12, 어두운 앰버 틴트)로 하이라이트(레퍼런스는 파란 테두리를 쓰지만 우리 앱은 브랜드 앰버로 통일)
- 하단 Primary CTA는 **현재 선택 상태를 반영한 동적 라벨**을 사용한다(예: "대수반 승인요청하기", "엄마는외계인 고르기"와 동일한 패턴) — 아무것도 선택하지 않은 경우 버튼은 비활성(회색) 처리하지 않고 "선택 후 활성화" 안내 텍스트를 버튼 하단에 작게 노출한다(비활성 버튼 남용 금지 원칙).

## 6. 팝업 화면의 버튼 구현
| 버튼 유형 | 배경 | 텍스트 | 사용처 |
|---|---|---|---|
| Primary (팝업 내 단일 CTA) | `--brand-amber` 채움, 코너 12px, 전체 너비 - 24px 여백 | 흰색, 15px/700 | "선물 받기", "승인요청하기", "OO 고르기" |
| Secondary (팝업 내 보조 액션) | 투명 + 1px `--border` | `--text-primary`, 14px/600 | "다음에 하기", "취소" |
| 팝업 닫기(X) | 원형 28px, 배경 `--surface-card` | 아이콘만 | 우상단 고정 |

버튼 높이는 팝업 내 CTA 기준 48~52px(일반 화면 버튼 44px보다 큼 — 팝업은 단일 결정 지점이므로 강조).

## 7. 내비게이션 바 (이전/바로가기 버튼)
레퍼런스 공통 패턴: 좌측 back 화살표(아이콘 전용) + 우측 텍스트형 유틸리티 링크.

- **Back 버튼**: 좌측 정렬, `‹` 또는 chevron-left 아이콘, 크기 20px, 탭 영역 44×44px, 색상 `--text-primary`. 클릭 시 이전 화면으로 복귀(모달의 경우 모달 닫기와 동일 동작).
- **바로가기/유틸리티 텍스트 버튼**: 우측 정렬, 13~14px/600, 색상 `--text-secondary`, 배경/보더 없음. 예: "안내", "자주하는 질문", "건너뛰기". 아이콘 없이 텍스트만 사용해 back 버튼과 시각적 위계를 다르게 둔다.
- 상단바 높이 48~52px, 좌우 패딩 16px, back 버튼과 타이틀 사이 8px.
- 모달/바텀시트에서는 back 버튼 대신 우상단 원형 X 버튼(6절 참고)을 사용하고, 좌측에는 필요시 타이틀만 배치한다.

## 8. 펼치기/접기(아코디언) 레이아웃
현재 앱의 "출석 기준 안내", "칭찬 요청 내역", "스티커 이력" 등에 적용된 패턴을 표준화한다(레퍼런스의 "더보기 ⌄" 패턴과 동일 구조).

- 헤더 행: 좌측 라벨(Subtitle-Micro 사이, 12.5px/700, `--text-secondary`) + 우측 chevron 아이콘(11px)
- 기본 상태: **접힘**(요약/불필요한 세부 정보는 기본적으로 숨김 — 정보 밀도를 낮춰 핵심 액션에 집중)
- 펼침 트랜지션: chevron `transform: rotate(180deg)`, 0.2s ease. 콘텐츠는 별도 높이 애니메이션 없이 즉시 노출(성능/단순성 우선), 상단 마진 8px
- 레퍼런스처럼 콘텐츠 중앙에 단독으로 배치되는 "더보기" 링크형(예: 카드 비교 화면)도 허용 — 이 경우 텍스트+chevron을 가운데 정렬하고 `--text-secondary` 컬러의 13px/600 사용.

## 9. 탭 / 세그먼트 컨트롤
두 가지 스타일을 용도에 따라 구분한다.
- **필터/스코프 전환(칩 탭)**: 랭킹 그룹 전환, 스티커 유형 전환 등 — 알약형(pill) 버튼, 선택 시 배경 `--brand-amber` 채움 + `--surface-page`(어두운) 텍스트로 강한 대비, 비선택 시 `--surface-card` 배경 + 1px `--border`.
- **컨텐츠 카테고리 전환(언더라인 탭)**: 레퍼런스의 "통합내역 | 자산분석"처럼 동등한 두 뷰를 전환할 때 — 하단 2px 밑줄 인디케이터, 선택 텍스트 `--text-primary`/700 + 인디케이터도 `--text-primary`, 비선택 `--text-muted`/400.

## 10. 가로 스크롤 카드(캐러셀)
상품(리워드) 슬라이드, 통계 비교 카드에 사용.
- 카드 폭 140~150px 고정, 카드 간 gap 10px, 좌우 첫/끝 카드 여백 16px
- 스크롤 스냅(`scroll-snap-type: x mandatory`) 적용 권장
- 카드 상단에 상태를 색으로 구분해야 하는 경우(예: Best/Worst 성과 비교) 카드 헤더에만 의미색을 사용하고 카드 전체 배경을 물들이지 않는다(가독성 유지)
- 카드 하단 CTA 버튼은 Secondary 스타일(44px 미만 높이 허용, 최소 36px)로 통일

## 11. 아이콘 · 뱃지 · Pill
- 상태 pill: 코너 999px(완전 라운드), 패딩 3px 10px, 폰트 11px/700
  - 대기: 배경 `--warning-bg` / 텍스트 `--warning-text`
  - 승인·정상: 배경 `--success-bg` / 텍스트 `--success`
  - 반려·취소: 배경 `--danger-bg` / 텍스트 `--danger`
  - 고정(공지): 배경 `--warning-bg` / 텍스트 `--warning-text`
- 등급 뱃지(원형): 26px, 금/은/동은 2절 색상표 사용, 4위 이하는 `--surface-card` 배경에 순위 숫자
- 아이콘은 라인(outline) 스타일 우선, 크기 18~20px 인라인 / 24px 데코레이티브 최대

## 12. 반응형 & 접근성
- 브레이크포인트: 모바일 ~430px(단일 컬럼) / 태블릿 431~1024px(카드 2~3열) / 데스크톱 1025px~(관리자 화면은 사이드바+메인 2컬럼 고정, 학생 화면은 중앙 정렬 최대 480px 폭 유지)

- 포커스 상태: 키보드 포커스 시 `box-shadow: 0 0 0 2px var(--brand-amber)` 링 적용
- 폼 요소(입력창/토글/버튼) 최소 높이 44px 유지, 토글(반 신청)은 크기 24×40px 이상 확보
